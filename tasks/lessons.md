# Lessons

## README and Release Copy

- Standard open source README pages should lead with what the project is, who it is for, what it can do, how to install it, and how to run it. Capability limits and safety boundaries should be present, but usually belong in a later "Scope and Safety Notes" section unless they are immediate hazards.
- Do not let release-page visual work push defensive or awkward wording into the README. The README is the evaluation path for users; keep the first screen direct, normal, and useful.
