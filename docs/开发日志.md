# 电池接管开发日志

## 项目概况
- 项目名称：电池接管
- 目标平台：macOS 15.x + Apple Silicon
- 核心目标：分钟级采集电池状态、按阈值控充、形成可复盘记录，并提供轻量桌面入口。

## 2026-03-03

### 立项与基线调研
- 明确产品目标不是“物理旁路电池”，而是在 macOS 可行范围内实现“停止继续充电并维持区间”。
- 完成开源方案与成型产品摸底，确定优先兼容 `battery` / `batt` 两类后端。
- 明确首发形态为 `CLI + LaunchAgent`，而不是菜单栏 App 或重 GUI。

### MVP 骨架落地
- 建立 Python + SQLite 项目结构。
- 完成 CLI 主命令：`doctor`、`init`、`sample`、`enforce`、`agent`、`status`、`report daily`。
- 建立三张核心表：`samples`、`actions`、`runtime_state`。
- 配置策略定为：
  - `>=92%` 停充
  - `<=88%` 恢复
  - 观察期 `24` 小时
  - 动作冷却 `300` 秒

## 2026-03-04

### 采集、策略、执行器联通
- 完成 `pmset -g batt` 与 `system_profiler SPPowerDataType` 解析。
- 完成 1 分钟采样、样本入库、日报生成。
- 完成策略引擎：
  - 观察期 `OBSERVE_ONLY`
  - 正常控充 `ACTIVE_CONTROL`
  - 异常降级 `DEGRADED_READONLY`
- 接入执行器路由，优先 `battery`，不可用时自动切换 `batt`。

### 首轮真实控充验证
- 完成观察期后切换到真实控充。
- 通过 `batt` 成功把上限设置到 `92%`。
- 验证 `batt status` 可见：
  - `Upper limit: 92%`
  - `Use power adapter: ✔`

### 运行形态升级
- 新增 `launchd` 安装脚本，将运行副本同步到 `~/Library/Application Support/BatteryTakeover/app`。
- 避免直接从桌面路径运行导致的权限与稳定性问题。
- 新增 `control.sh` 统一启动、停止、状态查看。

### 可视化面板与桌面入口
- 新增轻量 Dashboard：
  - 实时概览
  - 24h 曲线
  - 动作审计
  - 阈值修改
  - 立即执行策略
- 新增 `电池接管.app` 桌面入口，点击后自动启动服务并打开面板。

## 2026-03-05

### 桌面入口稳定性修复
- 定位到桌面 App 启动脚本对带空格路径处理错误，导致点击后浏览器打开但服务未真正启动。
- 修复 AppleScript 启动命令的路径引用与就绪等待逻辑。
- 重装桌面 App 后，点击动作可正确拉起 `agent + dashboard`。

### Dashboard 空白问题修复
- 定位到前端显示全 `-` 的根因不是前端渲染，而是后台 API 空响应。
- 追踪发现 `sqlite3.Connection` 未被可靠关闭，长期运行后触发 `Too many open files`。
- 修复存储层连接管理，改为显式 session 生命周期。
- 为 Dashboard API 增加异常兜底，为前端刷新增加错误提示，避免静默失败。

## 2026-05-05

### 安全与发布前审计收敛
- 按本地工具的真实风险面重新审计 Dashboard、采集器和发布流程。

## 当前已实现能力
- 电池信息采集与持久化
- 92/88 阈值控充
- 异常自动降级只读
- 24 小时日报复盘
- 开机自动运行
- 轻量桌面入口
- Web Dashboard 可视化与阈值编辑
- 发布页视觉预览
- README 中英双语切换

## 当前已知限制
- 不承诺“物理意义上的适配器直供绕过电池”。
- 控充能力依赖第三方后端，首发主要验证 `batt`。
- 安装包当前未做开发者签名与 notarization，首次安装可能触发 macOS 安全提示。
- App Store 上架暂不推进；当前更适合作为个人边缘工具和 GitHub 开源项目维护。

## 暂停开发时的保留事项
- 如继续面向更多用户分发，优先考虑 Developer ID 签名与 notarization，而不是直接上架 Mac App Store。
- 如后续恢复开发，优先做维护性增强：更清晰的异常提示、安装/卸载说明、后端依赖诊断。
- 不扩大 Dashboard 绑定范围，除非同时加入明确的认证边界。
