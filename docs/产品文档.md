# 电池接管产品文档

## 1. 产品概述

### 1.1 产品名称
电池接管

### 1.2 产品定位
一个运行在 macOS 上的电池监控与阈值控充工具，用于长期插电办公场景下减少无谓满充，并让用户能够追踪和复盘电池行为。

### 1.3 核心价值
- 用分钟级数据记录电池真实状态，而不是靠体感判断。
- 用明确阈值控制电池上充区间，减少长期 100% 满充停留。
- 出现异常时自动降级为只读，不盲目继续执行写操作。
- 提供轻量桌面入口，降低日常使用门槛。

## 2. 问题定义

### 2.1 用户问题
在长期接入电源办公时，很多用户不希望电池反复维持在 100%，但 macOS 原生能力对细粒度电池策略控制有限。

### 2.2 本产品要解决的事
- 查询并记录电池、电源、健康状态。
- 在兼容后端可用的前提下，按用户设定阈值进行控充。
- 把所有样本与动作沉淀为可复盘证据。

### 2.3 本产品不承诺的事
- 不承诺物理层面的“完全绕过电池直供主板”。
- 不承诺对所有 macOS 版本、所有芯片和所有第三方后端百分百兼容。

## 3. 目标用户
- 长期插电办公的 Mac 用户
- 对电池寿命和行为有明确管理需求的个人用户
- 希望保留可审计记录，而不是只依赖一个“限充开关”的用户

## 4. 使用场景
- 日常桌面办公，MacBook 长时间连接适配器
- 用户希望电量在例如 `88% - 92%` 区间内波动
- 用户需要回看过去一天的充电、停充和异常记录

## 5. 产品范围

### 5.1 当前版本已包含
- CLI 命令行工具
- SQLite 本地存储
- 采样、阈值策略、动作审计
- LaunchAgent 开机自动运行
- Dashboard 可视化面板
- 桌面 App 点击入口

### 5.2 当前版本不包含
- 菜单栏 GUI
- 云同步和多设备协同
- 复杂预测模型
- 非 macOS 平台支持

## 6. 功能说明

### 6.1 体检与初始化
- `doctor`：检查系统命令、执行器可用性和基础运行条件。
- `init`：初始化目录、数据库和运行态。

### 6.2 数据采集
- 默认每 60 秒采集一次。
- 采集来源：
  - `pmset -g batt`
  - `system_profiler SPPowerDataType`
- 核心字段：
  - 电量百分比
  - 是否插 AC
  - 是否处于充电态
  - 循环次数
  - 最大容量百分比

### 6.3 控充策略
- 停充阈值：默认 `92%`
- 恢复阈值：默认 `88%`
- 策略具有滞回区间，防止在边界来回抖动。
- 带最小动作间隔，避免频繁下发。

### 6.4 运行模式
- `OBSERVE_ONLY`
  - 只采集，不执行真实控充
  - 默认用于前 24 小时观察
- `ACTIVE_CONTROL`
  - 执行真实阈值控充
- `DEGRADED_READONLY`
  - 出现连续失败或关键后端不可用时启用
  - 保留采集与告警，停止继续写控充动作

### 6.5 可视化面板
- 当前电量
- 当前模式
- Agent 状态
- 24 小时样本/动作统计
- 24 小时电量曲线
- 最近动作列表
- 阈值编辑与立即执行

### 6.6 桌面入口
- 安装后生成 `电池接管.app`
- 点击后自动拉起服务并打开本地面板
- 目标是“日常使用不需要频繁打开终端”

## 7. 技术架构

### 7.1 技术栈
- Python
- SQLite
- launchd / LaunchAgent
- 简单内置 HTTP Dashboard

### 7.2 核心模块
- `collector.py`
  - 负责系统命令调用与原始数据解析
- `policy.py`
  - 负责阈值决策、滞回和降级状态机
- `storage.py`
  - 负责 SQLite 读写
- `agent.py`
  - 负责循环执行、采样、策略评估与写动作
- `dashboard.py`
  - 负责可视化面板与配置接口
- `executors/`
  - 负责封装 `battery` / `batt` / `noop`

## 8. 运行依赖
- macOS 15.x
- Apple Silicon
- Python 3.11+
- 可用的 `batt` 或 `battery` 后端
- 对 `batt` 而言，daemon 必须正常运行且权限配置正确

## 9. 安装与启动

### 9.1 开发者安装
```bash
git clone <repo-url>
cd battery-takeover
python3 -m venv .venv
source .venv/bin/activate
pip install -e .
```

### 9.2 基础检查
```bash
./btake --config ./config/default.toml doctor
./btake --config ./config/default.toml init
```

### 9.3 启动方式
```bash
./control.sh start
./control.sh status
```

### 9.4 开机自启
```bash
./install_agent_launchd.sh
```

### 9.5 安装桌面入口
```bash
./install_desktop_app.sh
```

## 10. 安全与风险控制

### 10.1 风险来源
- 后端命令失效
- macOS 更新引入兼容性变化
- 第三方 daemon 权限异常
- 采集命令超时或输出格式变化

### 10.2 当前缓解机制
- 每次运行先做后端探测
- 失败时自动降级只读
- 动作与原始样本保留入库，便于回放和诊断
- Dashboard 与日志同时提供错误信号
