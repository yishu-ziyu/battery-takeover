# 调研：macOS 电池控制的开源与产品基线（2026-03-03）

## 1) 候选基线清单

> 成熟度快照（2026-03-03）：以下 star/版本信息来自各项目公开页面与仓库检索结果，属于调研时点快照，后续可能变化。

### A. ActuallyMentor / battery（开源）
- 类型：CLI 工具（支持设置充电上限）
- 成熟度：约 6.8k stars，近一年仍有版本发布
- 适合场景：命令行驱动、可脚本化、便于与你的“复盘系统”集成
- 优势：社区使用广、文档和安装路径清晰
- 风险：依赖底层机制，macOS 新版本可能出现兼容波动

### B. finnvoor / batt（开源）
- 类型：Homebrew 安装的电池管理工具（可设置充电上限）
- 成熟度：约 1.6k stars，近一年仍有版本发布
- 适合场景：希望保留较轻量 CLI 能力，便于快速验证
- 优势：安装与使用门槛低
- 风险：需要重点验证在你的目标机型与系统上的稳定性

### C. zackelia / bclm（开源，较早方案）
- 类型：Battery Charge Level Max 工具
- 成熟度：历史项目，约 2.8k stars
- 优势：经典、思路清晰
- 明显限制：官方 README 已提示在 macOS 15+ 受 SIP 机制影响，不再按过去方式可用

### D. AlDente（成型产品）
- 类型：成熟商业产品（含免费版）
- 适合场景：你需要先用现成产品验证“策略体验”，再做自研
- 优势：产品化完整、功能覆盖全面（充电限制/状态可视化等）
- 风险：核心能力非开源，不适合作为“可二次开发代码基线”

### E. BatFi（成型产品）
- 类型：Mac App（有开源仓库）
- 成熟度：约 300+ stars
- 适合场景：希望参考 UI 与用户流程，或评估是否直接使用成品
- 风险：作为“二开底座”前需评估仓库活跃度与架构可扩展性

## 2) 结论：建议作为开发基础板的优先级

1. **首选基线：ActuallyMentor / battery**
- 原因：开源、CLI 友好、最适合做“策略引擎 + 复盘系统”的可编排底座。

2. **备选基线：finnvoor / batt**
- 原因：可作为兼容性和稳定性的对照组，便于快速 AB 验证。

3. **产品对照：AlDente / BatFi**
- 原因：用于反向抽取产品能力清单（我们该做什么），不建议作为代码底座。

## 3) 建议的技术路线（MVP）

### 阶段一：监控与复盘（只读，不改系统行为）
- 采集：`pmset -g batt` + `system_profiler SPPowerDataType`。
- 存储：本地 SQLite（分钟级快照 + 事件表）。
- 输出：每日/每周复盘报告（充电区间、峰值、低电风险、插拔频率）。

### 阶段二：阈值控充执行器
- 基于开源 CLI（优先 `battery`）做策略执行：
  - 示例策略：`>=92%` 时设置/保持“停止继续充电”；
  - 低于回落阈值（如 88%）再恢复充电，避免频繁抖动（滞回）。
- 通过 LaunchAgent 定时巡检与补偿执行。

### 阶段三：产品化增强
- 菜单栏状态展示 + 策略一键切换。
- 异常告警（例如策略失效、系统更新后权限变化）。
- 数据可视化复盘（7 天/30 天视图）。

## 4) 风险前置
- Apple Silicon 与 Intel 的底层能力差异需要分流实现。
- 新 macOS 版本可能影响 SMC/权限路径，必须有“失效检测 + 安全降级”。
- “适配器优先供电”应在文案中准确表达为“停止继续充电并维持电量区间”，避免过度承诺。
